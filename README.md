# EccommerOracle 🛒

A full-stack e-commerce back-office management system built with **Python Flask** and **Oracle XE**. Designed to demonstrate real-world relational database features — including triggers, constraints, cascading deletes, and an audit log — through a clean, web-based admin interface.

---

## Features

| Module | Description |
|--------|-------------|
| **Dashboard** | Live KPI snapshot — total customers, products, orders, and revenue. Highlights low-stock items. |
| **Customers** | Full CRUD — register, update, and delete customer profiles. |
| **Products** | Manage product catalog and inventory levels. |
| **Orders** | Create orders, add line items, and track status (PENDING → PAID → SHIPPED). |
| **Payments** | Record payments against pending orders. Automatically marks orders as PAID via DB trigger. |
| **Audit Logs** | Admin-only view of all system events generated by Oracle triggers. |
| **Auth** | Session-based login with role-based access control (`ADMIN` / `STAFF`). |

---

## Database Triggers

Three Oracle triggers drive the core business logic automatically at the database level:

- **`trg_update_stock`** — Deducts product inventory on every new order item insert. Rolls back if stock would go negative.
- **`trg_calculate_order_total`** — Recalculates the order's `total_amount` on every item addition.
- **`trg_payment_auto_status`** — Flips the order status to `PAID` when a payment record is inserted.

## Tech Stack

- **Backend**: Python 3, Flask
- **Database**: Oracle Database XE (via `python-oracledb`)
- **Frontend**: Jinja2 templates, vanilla CSS
- **Config**: `python-dotenv`

## Getting Started

### Prerequisites
- Python 3.10+
- Oracle Database XE (or Oracle Cloud Free Tier)
- Oracle Instant Client (if using thick mode)

### 1. Clone the repository
```bash
git clone https://github.com/Kimchi991/EccommerOracle.git
cd EccommerOracle
```

### 2. Set up environment variables
Copy the example file and fill in your Oracle credentials:
```bash
cp .env.example .env
```

Edit `.env`:
```env
ORACLE_USER=your_oracle_username
ORACLE_PASSWORD=your_oracle_password
ORACLE_DSN=localhost/XEPDB1
FLASK_SECRET_KEY=your_secret_key
FLASK_PORT=5000
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Initialize the database
Run the schema script against your Oracle instance (via SQL*Plus or SQL Developer):
```sql
@schema.sql
```
Or use the Python setup script:
```bash
python setup_db.py
```

### 5. Run the app
```bash
python app.py
```

Navigate to `http://127.0.0.1:5000`. Log in with one of the seeded accounts:

| Username | Password | Role |
|----------|----------|------|
| `admin` | `admin123` | ADMIN |
| `manager` | `manager123` | ADMIN |
| `staff1` | `staff123` | STAFF |

## Security Notes

- `.env` is excluded from version control via `.gitignore`. **Never commit real credentials.**
- Passwords in the seed data are plaintext for academic/demo purposes. Use hashing (e.g., `bcrypt`) in any production deployment.
- Delete operations on sensitive records are restricted to `ADMIN` role only.
